==Multiple Ports==
These are the same as a single port foward however if you have more than 3 ports needing forwarded you will have to go in and manually change the rule number to 4,5, etc
For example: if they need port 88 and port 94 both forwarded to 192.168.5.7 then your two command are as follows:
'''nat port-forwarding add 1 88 88 88 88 both 192.168.5.7 0.0.0.0'''
'''nat port-forwarding add 2 94 94 94 94 both 192.168.5.7 0.0.0.0'''

==Port Range==
2nd difference) If it's a range of ports then instead of repeating the port number 4 times you'll actually include the port range and repeat that twice.
For example if the customer would like ports 8870 through 8925 to be forwarded to local ip 192.168.44.13 then your command would look like this:
nat port-forwarding add 8870 8870 8925 8870 8925 both 192.168.44.13 0.0.0.0

==Port Masking== (3rd difference) If our customer needs to have their external port go to a different internal port, IE for port masking/hiding the internal port from the outside world we will need to enter both the external the internal ports into the rule.
For example if the customer would like port 3500 forwarded to 3389 internally to local ip 192.168.44.57 then your command would look like this:
nat port_forwarding add 2 3500 3500 3389 3389 both 192.168.44.57 0.0.0.0
